# 脚本职责树与关键数据流

## 战斗主控

- `scripts/main.gd`：战斗场景主控和 flow 装配。
- `scripts/game/game_scene_wiring.gd`：创建并连接 HUD、升级 UI、暂停菜单、角色面板等 UI。
- `scripts/game/game_hud_flow.gd`：把战斗状态投影到 HUD。
- `scripts/game/game_character_panel_flow.gd`：角色面板打开、关闭和暂停规则。
- `scripts/game/blessing_unlock_notice_flow.gd`：技能解锁/进化消耗祝福材料时的短提示弹窗。
- `scripts/game/reward_flow.gd`：升级、小 Boss 奖励和最终奖励流程。
- `scripts/game/enemy_spawn_flow.gd`：波次刷怪和刷新警告。
- `scripts/game/enemy_defeat_flow.gd`：敌人死亡后的奖励、成就和 Boss 流程。

## 玩家模块

- `scripts/player.gd`：玩家节点协调层。
- `scripts/player/player_blessing_system.gd`：祝福定义、发牌、应用和合成。
- `scripts/player/player_blessing_skill_state.gd`：祝福驱动的技能解锁和技能倍率。
- `scripts/player/player_blessing_skill_bridge.gd`：玩家节点上的祝福查询、合成、绑定选择和技能事件提示桥接。
- `scripts/player/player_level_flow.gd`：升级选项、英雄特性和小 Boss 奖励入口。
- `scripts/player/player_upgrade_applier.gd`：升级/奖励分发器。
- `scripts/player/player_equipment_flow.gd`：装备奖励、角色持有、转赠和装备数值。
- `scripts/player/player_skill_cooldown_flow.gd`：技能栏槽位数据。
- `scripts/player/player_timer_flow.gd`：玩家运行时计时器、临时 buff、技能冷却更新和开发者无冷却清理。
- `scripts/player/player_authored_effects.gd`：带动画场景的技能特效创建、播放、复用和释放。
- `scripts/player/player_effect_primitives.gd`：线条、圆环、爆发、护盾、文字等轻量特效创建、复用和释放。
- `scripts/player/player_run_save_state.gd`：运行中存档。
- `scripts/player/roles/*_role.gd`：三名角色普攻和角色差异。
- `scripts/abilities/*_ability.gd`：祝福技能的独立行为。

## UI 模块

- `scripts/level_up_ui.gd`：升级和奖励菜单。
- `scripts/hud.gd`：战斗 HUD。
- `scripts/ui/hud/character_panel.gd`：角色面板、祝福、合成和道具展示。
- `scripts/ui/hud/combat_skill_bar.gd`：底部技能栏。
- `scripts/ui/components/*`：卡片、悬停详情和通用 UI 组件。

## 升级数据流

`player.gd`
-> 获得经验并满足升级条件
-> `player_level_flow.build_blessing_upgrade_options`
-> `player_blessing_system.build_offer_for_owner`
-> `level_up_ui.gd` 展示祝福
-> 玩家选择
-> `player_upgrade_applier.apply_upgrade`
-> `player_blessing_system.apply_option`
-> 更新祝福状态、刷新技能解锁、刷新 HUD 和角色面板

## 祝福技能数据流

祝福选择
-> `player_blessing_system.gd` 记录角色祝福或技能祝福
-> `player_blessing_skill_state.gd` 检查技能解锁与进化，并在锁定配方材料的事件中写入 `material_lines`
-> `player_blessing_skill_bridge.gd` 处理绑定选择队列、战斗文字和 `blessing_skill_event_announced` 信号
-> `blessing_unlock_notice_flow.gd` 显示“祝福材料已消耗”短提示
-> `player_skill_cooldown_flow.gd` 生成技能栏槽位
-> 对应 `scripts/abilities/*_ability.gd` 在战斗中触发

## 小 Boss 奖励数据流

小 Boss 死亡
-> `enemy_defeat_flow.gd`
-> `reward_flow.gd`
-> `player_level_flow.get_small_boss_reward_options`
-> `player_equipment_flow.get_active_reward_options`
-> `level_up_ui.gd`
-> `player_upgrade_applier.apply_upgrade`
-> `player_equipment_flow.apply_equipment_reward`

## 存档数据流

`main.gd`
-> `save_manager.gd`
-> `player_run_save_state.gd`
-> 保存角色状态、祝福状态、技能祝福、装备和战斗进度。

读取运行中存档时，`player_run_save_state.gd` 先恢复角色列表、装备、属性训练和祝福状态，再按最终角色最大生命值归一化 `role_health_values`。这样带有祝福/装备生命加成的角色不会在读档时被默认生命上限错误截断。

## 开发护栏

- 新祝福先改 `player_blessing_system.gd`。
- 新祝福技能先改 `player_blessing_skill_state.gd`，再加 ability。
- 新奖励先判断是 `player_reward_applier.gd`、`player_equipment_flow.gd` 还是 `reward_flow.gd`。
- 新 UI 不直接堆进 `main.gd` 或 `player.gd`。
- 升级数据流以祝福体系为准。
