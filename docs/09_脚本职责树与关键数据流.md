# 脚本职责树与关键数据流

## 一、脚本职责树

### 1. 战斗主控层

- `main.gd`
  负责战斗主场景的总控
  主要职责：
  - 找到玩家节点
  - 装配 HUD / 升级菜单 / 暂停菜单 / 失败界面
  - 控制生存时间、关卡进度、刷怪、精英、Boss
  - 处理继续游戏、自动存档、主线关卡上下文
  - 在战斗中持续刷新 HUD

- `scripts/game/game_achievement_bridge.gd`
  负责把战斗运行时事件翻译成本地成就事件。
  交接边界：
  - `main.gd` 只调用这个桥接层，不直接调用 `AchievementService`
  - 桥接层只知道本地 `AchievementService`
  - Steam/GodotSteam 同步不允许写进这里

- `scripts/game/enemy_defeat_flow.gd`
  负责敌人死亡后的战斗流程后果。
  当前包含：
  - 记录击杀类成就
  - 开发者模式 Boss 死亡清理
  - 精英死亡 HUD 刷新
  - 小 Boss 奖励菜单触发
  - 无尽 Boss 计数、成就、Boss UI 隐藏与奖励菜单触发
  - 普通 Boss 死亡后进入最终核心奖励流程

- `scripts/game/game_scene_wiring.gd`
  负责战斗场景内 UI 节点创建与信号连接。
  当前包含：
  - HUD 实例化与开发者模式信号连接
  - 角色面板实例化与关闭信号连接
  - 升级 UI、暂停菜单、结算 UI 的信号连接
  - 玩家经验、升级、属性、血量、法力、死亡信号连接

- `scripts/game/game_story_context_flow.gd`
  负责战斗场景使用的故事/无尽上下文。
  当前包含：
  - 读取当前故事关卡
  - 判断故事模式与无尽模式
  - 应用故事档中的编队和样式
  - 派生 Boss 出场时间、刷怪曲线时间、刷怪间隔倍率、敌人血量/速度倍率

- `scripts/game/game_hud_flow.gd`
  负责把战斗状态投影到 HUD。
  当前包含：
  - 常规 HUD 全量刷新
  - 每帧时间、统计、Boss HUD 刷新
  - 玩家经验、属性、血量、法力回调对应的 HUD 更新
  - Boss UI 显示/隐藏
  - 开发者/无尽模式性能指标采样显示

- `scripts/game/game_character_panel_flow.gd`
  负责角色面板显示/隐藏和暂停规则。
  当前包含：
  - 判断角色面板是否可打开
  - 打开角色面板并暂停战斗
  - 关闭角色面板并恢复暂停状态
  - 处理角色面板按键切换入口

- `scripts/game/game_map_flow.gd`
  负责地图展示与小地图数据投影。
  当前包含：
  - 创建世界空间地图边界节点
  - 配置 HUD 小地图边界
  - 收集玩家、敌人、Boss、经验晶体、回复道具位置
  - 每帧把小地图 payload 推送给 HUD

- `scripts/map/map_boundary_view.gd`
  负责绘制世界空间地图边界。

- `scripts/player/player_map_bounds_flow.gd`
  负责玩家移动后的边界钳制。

### 2. 玩家战斗层

- `player.gd`
  当前项目最核心脚本
  主要职责：
  - 玩家移动与面朝方向
  - 三角色切换与队伍顺序
  - 普攻逻辑
  - 进场技 / 退场技 / 后台协同
  - 大招释放与后续收益
  - 经验、升级、等级成长
  - Build 候选池、Build 结果应用
  - 英雄特性训练
  - 角色风格样式
  - 与 HUD 的数据输出
  - 大量战斗特效和命中逻辑

- `scripts/player/player_attribute_flow.gd`
  负责英雄特性训练数据、旧存档迁移、共同致富计数、剑士最大生命同步。

- `scripts/player/roles/role_attribute_rules.gd`
  负责英雄特性数值规则：剑士生命/回复、枪手移速/闪避、术师回能/拾取范围、对应英雄普攻伤害、共同致富切人冷却系数。

- `scripts/player/player_theme_skill_flow.gd`
  负责主题 Build 触发型效果和独立冷却被动：起潮、贯流/分浪/环潮、退潮、蓄相/破相/归一等。

- `scripts/player/player_skill_cooldown_slots.gd`
  负责 HUD 技能栏槽位 payload：普攻槽、独立冷却被动槽，以及普攻强化说明折叠。

- `scripts/player/player_ultimate_flow.gd`
  负责大招消耗、释放、当前英雄大招显示文案，以及大招强化说明折叠。

### 3. 敌人与弹幕层

- `enemy.gd`
  负责普通怪、精英、Boss 的基础行为
  主要职责：
  - 敌人移动与追击
  - 远程怪、加速怪、冲刺怪等行为模式
  - 状态异常：减速、易伤、流血
  - Boss 阶段切换与招式逻辑
  - 敌人被击败后的掉落与回报

- `enemy_bullet.gd`
  负责敌方弹幕实体逻辑

- `bullet.gd`
  负责我方子弹实体逻辑，主要服务枪手与部分技能

### 4. UI 层

- `hud.gd`
  负责战斗信息 UI
  主要职责：
  - 时间、等级、经验
  - HP / MP / 符印
  - 队伍面板
  - 切人 CD / 接力提示
  - Boss 血条

- `level_up_ui.gd`
  负责升级 / Build 菜单
  主要职责：
  - 英雄特性训练 + Build 卡双选择
  - 卡牌短摘要列表
  - 悬停详情浮层
  - 小 Boss 道具/卡牌双选
  - 最终效果卡进度展示

- `scripts/ui/core/survivors_modal.gd`
  统一全屏遮罩与居中弹窗，自适应窗口尺寸。

- `scripts/ui/components/survivors_card_list.gd`
  统一卡牌列表/网格、常显滚动条、短摘要和选中态。

- `scripts/ui/components/survivors_hover_detail.gd`
  统一悬停详情浮层，支持内容自适应和三英雄效果去重。

- `scripts/ui/hud/combat_skill_bar.gd`
  负责底部切人/普攻/独立被动/大招能量/经验条显示，并把说明交给 hover detail。

- `pause_menu.gd`
  负责 ESC 菜单

- `game_over_ui.gd`
  负责失败结算与重开入口

### 5. 菜单与主线流程层

- `main_menu.gd`
  负责主菜单逻辑

- `save_select.gd`
  负责存档位选择

- `story_prep.gd`
  负责主线准备界面 / 功能版营地

- `story_data.gd`
  负责主线关卡表、角色池、风格样式等静态数据

### 6. 基础服务层

- `save_manager.gd`
  负责战斗存档、故事存档、继续游戏目标

- `achievements/achievement_service.gd`
  本地成就服务，负责读取成就定义、保存本地进度、发出解锁信号

- `achievements/achievement_notifier.gd`
  本地成就弹窗，只监听解锁信号，不参与解锁判定

- `achievements/steam_achievement_adapter.gd`
  后续 Steam 发布时的同步层，只监听本地解锁信号并调用平台 API

- `bgm_player.gd`
  负责 BGM、音量配置、静音、续播位置

- `developer_mode.gd`
  负责开发者模式总开关

## 二、关键数据流

### 1. 启动流程

`project.godot`
-> `main_menu.tscn`
-> `main_menu.gd`
-> 选择：
- 主线模式 -> `save_select.tscn`
- 继续游戏 -> 上次继续目标场景
- 开发者模式 -> `main.tscn`

### 2. 主线流程

`save_select.gd`
-> 选择存档位
-> `save_manager.gd` 建立或读取故事档
-> `story_prep.tscn`
-> `story_prep.gd`
-> 进入下一关
-> `main.tscn`
-> `main.gd`
-> `game_story_context_flow.gd` 读取当前关卡上下文并应用故事配队

### 3. 战斗内核心数据流

`main.gd`
-> 初始化 `player`
-> 读取故事关卡上下文
-> 通过 `game_scene_wiring.gd` 创建 UI 并连接玩家信号
-> 通过 `game_map_flow.gd` 创建地图边界并配置小地图
-> 定时刷新 HUD

`player.gd`
-> 战斗中触发：
- `experience_changed`
- `level_up_requested`
- `stats_changed`
- `health_changed`
- `mana_changed`
- `died`

`main.gd`
接收这些信号后：
- 通过 `game_hud_flow.gd` 更新 HUD
- 打开升级菜单
- 处理死亡与结算

敌人死亡信号：

`enemy.gd`
-> `defeated(enemy_kind)`
-> `main.gd._on_enemy_defeated`
-> `scripts/game/enemy_defeat_flow.gd`
-> 按敌人类型转发到成就、HUD、奖励或关卡完成流程

### 4. 升级 / Build 流程

`player.gd`
-> 判断获得经验后是否升级
-> 生成 Build 候选
-> 发出 `level_up_requested`

`main.gd`
-> 打开 `level_up_ui`

`level_up_ui.gd`
-> 玩家选择“英雄特性训练”和 Build 卡
-> 两者都选定后发出 `upgrade_selected(option_id, attribute_option_id)`

`main.gd`
-> 把结果回传给 `player.gd`

`player.gd`
-> 应用英雄特性成长 / Build 成长
-> `player_attribute_flow.gd` 更新特性、共同致富、切人冷却倍率
-> `player_build_progress_flow.gd` 记录卡牌等级并检测新主题解锁
-> 更新角色战斗状态

### 5. 存档与继续流程

`main.gd`
-> 每隔一段时间自动调用 `save_manager.gd`
-> 保存当前战斗状态

退出或切回菜单后：
- `main_menu.gd` 查询 `save_manager.gd`
- 如果有继续目标，显示“继续游戏”

继续游戏时：
- 回到 `main.tscn`
- `main.gd` 尝试恢复运行状态
- BGM 按记录位置附近恢复

### 6. 成就数据流

战斗事件：

`main.gd`
-> `scripts/game/game_achievement_bridge.gd`
-> `/root/AchievementService`
-> 本地保存 `user://achievements.json`
-> 发出 `achievement_unlocked`

显示与平台同步：

`AchievementNotifier`
-> 监听 `achievement_unlocked`
-> 显示右上角本地弹窗

`steam_achievement_adapter.gd`
-> 后续 Steam 版本启用
-> 监听 `achievement_unlocked`
-> 调用 GodotSteam `setAchievement/storeStats`

## 三、当前工程里的实际“核心枢纽”

当前最关键的 3 个枢纽脚本：

1. `player.gd`
   真正的战斗核心

2. `main.gd`
   战斗流程与系统总控

3. `save_manager.gd`
   主线 / 继续游戏 / 存档位的连接枢纽

## 四、继续开发时最该注意的地方

- 新功能尽量先判断应该挂在：
  `main.gd`
  还是 `player.gd`
  还是单独服务脚本
- 不要继续把“纯 UI 逻辑”塞进 `player.gd`
- 不要把“纯数据定义”继续硬写进大量 `match` 分支里
- 不要从 `main.gd` 直接调用平台 SDK 或全局业务服务；优先通过 `scripts/game/*` flow / bridge 隔离
- 不要把敌人死亡后的奖励分支重新塞回 `main.gd`；新增击败后果应先扩展 `enemy_defeat_flow.gd`
- 不要把新增 UI 或信号连接细节继续堆在 `main.gd`；新增战斗场景 wiring 应先扩展 `game_scene_wiring.gd`
- 不要把故事/无尽派生参数继续散落在 `main.gd` 或刷怪 flow；新增关卡上下文规则应先扩展 `game_story_context_flow.gd`
- 不要从其他 flow 直接调用 `hud.show_boss_ui` / `hud.hide_boss_ui` 等投影方法；优先扩展 `game_hud_flow.gd`
- 不要把角色面板打开/关闭和暂停规则写回 `main.gd`；优先扩展 `game_character_panel_flow.gd`
- 不要把地图边界/小地图数据采集散写在 `main.gd` 或 HUD；地图数据采集归 `game_map_flow.gd`，绘制归 `hud.gd` / `map_boundary_view.gd`

### 7. 主题 Build 数据流

`build_database.gd`
-> 定义主题、配比、卡牌、三英雄卡名、三英雄数值说明
-> `build_system.gd`
-> 根据已选卡等级和奖励等级生成当前卡池
-> `player_build_progress_flow.gd`
-> 选卡后记录等级并宣布新主题
-> `player_theme_skill_flow.gd`
-> 战斗命中/受击时按卡等级触发实际效果
-> `player_stat_payload.gd`
-> 把普攻、独立被动、大招说明投影给 HUD
-> `combat_skill_bar.gd` / `survivors_hover_detail.gd`
-> 显示短标题和悬停详情
