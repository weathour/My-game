# UI、菜单、存档与音频

## 主菜单

主菜单场景：
`res://scenes/main_menu.tscn`

当前主菜单功能：

- 继续游戏
- 主线模式
- 挑战模式（当前禁用）
- 无尽模式（当前已启用，进入 `res://scenes/endless_save_select.tscn`）
- 开发者模式
- 设置
- 退出

## 主菜单视觉规则

- 当前首屏即主菜单
- 背景图需要随窗口和全屏自适应拉伸
- 当前主菜单背景资源来自 `assets/demo2.png`

## 设置菜单

主菜单和暂停菜单都支持：

- 背景音乐音量滑条
- 背景音乐静音开关
- 性能日志记录开关（默认关闭，仅用于短时间排查卡顿）

主菜单设置面板实现：

- `scripts/main_menu.gd` 点击“设置”后调用 `settings_panel.open()`。
- `scripts/ui/main_menu/main_menu_settings_panel.gd` 根节点必须是全屏 `Control`，不能再依赖 `CenterContainer` 居中。
- 设置面板内部使用 `scripts/ui/core/survivors_modal.gd` 统一弹窗；打开和窗口尺寸变化时会重新 `apply_layout()`，保证弹窗居中且不贴角落。


## 显示设置

当前主菜单设置面板包含显示设置页：

- 窗口模式：窗口 / 全屏
- 窗口大小：`1280x720`、`1600x900`、`1920x1080`
- 窗口可拖拽调整，`WindowDisplayManager` 会将窗口尺寸校正为 `16:9`
- 项目默认使用 `canvas_items + keep`，避免非标准比例下画面被拉伸变形
- `project.godot` 显式启用 `window/size/resizable=true` 和 `window/stretch/aspect="keep"`，本地检查会校验这些显示配置。
- `project.godot` 显式设置 `run/max_fps=120` 且关闭 VSync 帧率钳制，避免渲染被固定在 60 FPS，同时把最高渲染帧率限制在 120 FPS；物理 tick 仍按 Godot 默认 60Hz 运行。
- 显示设置保存于 `user://settings.cfg` 的 `display` section
- 性能日志记录设置保存于 `user://settings.cfg` 的 `gameplay/performance_trace_enabled`；开启后写入 `user://performance_trace_latest.jsonl`，包含周期样本和慢帧样本。

## HUD

当前 HUD 方向：

- 常驻显示：时间、等级
- 核心显示：经验条、HP、MP / 大招能量
- Boss 出现时顶部切换为 Boss 名和 Boss 血条
- 底部战斗栏显示切人冷却、普攻冷却、独立冷却被动、大招能量球和经验条
- 技能/普攻/大招图标标题显示名称，不显示剩余冷却；剩余冷却和完整说明放在鼠标悬停详情中
- 大招能量球显示当前站场英雄的大招名称，悬停时显示该英雄大招和已获得大招强化
- 右下角显示小地图，包含玩家、敌人、Boss、经验晶体和回复道具位置
- 战斗栏表现刷新以 30 FPS 为目标，覆盖时间、切人冷却、技能冷却、大招能量和小地图位置刷新。
- 战斗地图边界在世界空间以蓝色边框显示，玩家不能越过边界
- 高级信息可通过按键切换显示
- 开发者性能信息包含 FPS、帧时间 p95 / p99 / max、自动存档 SavePeak、运行时实体计数、新怪入场队列 SpawnQ、保存峰值规模、敌人/敌方弹幕/掉落物批处理状态与峰值计数，便于在高密度战斗或长时间游玩中对比优化效果；开启“记录性能日志”后会额外把周期样本和 `>=33.4ms` 慢帧样本写入 JSONL 日志。

## 地图边界与小地图

当前战斗地图范围由 `scripts/main.gd` 的 `map_bounds` 导出属性控制，默认：

```gdscript
Rect2(Vector2(-1600.0, -900.0), Vector2(3200.0, 1800.0))
```

职责分配：

- `scripts/game/game_map_flow.gd`：创建地图边界显示，收集小地图位置数据
- `scripts/map/map_boundary_view.gd`：绘制世界空间边界框
- `scripts/player/player_map_bounds_flow.gd`：玩家移动后钳制在地图范围内
- `scripts/hud.gd`：绘制右下角小地图

小地图颜色约定：

- 青色：玩家
- 红色：普通敌人 / Boss
- 黄色：精英
- 粉色：小 Boss
- 绿色：经验晶体
- 粉红：回复道具

## 暂停菜单

触发方式：`ESC`

当前暂停菜单功能：

- 继续游戏
- 重新开始
- 返回主菜单
- 音量调节
- 静音开关
- 性能日志记录开关

## 性能日志

- 开关入口：主菜单设置或暂停菜单的“记录性能日志”。
- 默认关闭，建议只在复现卡顿前开启，复现几秒后关闭。
- 日志路径：`user://performance_trace_latest.jsonl`，每次重新开始记录会覆盖上一份。
- 每行是一个 JSON 对象；`reason=periodic` 是固定间隔样本，`reason=slow_frame` 是单帧耗时不低于 `33.4ms` 的慢帧样本。
- 样本包含：FPS、`delta_ms`、生存时间、实体计数、待刷怪队列、批处理状态、帧时间分位数、当前帧/峰值 counters、SavePeak 和性能 feature flags。

## 存档结构

核心脚本：
`scripts/save_manager.gd`

存档根目录：
`user://story_slots`

当前存档结构：

- `meta.json`
  记录最近使用的存档位等元信息

- `slot_x/story_profile.json`
  主线存档资料

- `slot_x/run_save.json`
  战斗中的继续存档

- `slot_x/run_save_backup.json`
  战斗存档备份

自动存档性能验证：

- 正常战斗会按 `scripts/main.gd` 的 `autosave_interval` 周期自动写入，当前默认约 2 秒一次。
- 升级、祝福绑定和技能奖励确认后的补充保存会延后到后续帧，与 HUD 完整刷新合并调度，避免奖励点击同一帧同时承担发牌、应用奖励、完整 HUD 文案重建和保存写入。
- 战斗存档固定覆盖当前存档位的 `run_save.json` 和 `run_save_backup.json`，不会按时间追加历史快照，因此磁盘文件数量不会随游玩时间增长。
- 战斗存档内容包括生存时间、BGM 播放位置、已刷精英/小 Boss/Boss 计数、Boss 击败计数、玩家运行态，以及当前敌人、敌方弹幕、经验晶体和回复道具的运行态数组。
- 性能面板会显示 `SavePeak` 与保存峰值规模（敌人、敌方弹幕、经验晶体、回复道具、payload KB），用于排查“玩着玩着开始掉帧”的存档尖峰。
- 自动化验证仍可通过性能 feature flag `disable_autosave` 做无 UI 对照；例如设置环境变量 `MY_GAME_PERF_DISABLE_AUTOSAVE=1` 后运行游戏，可临时跳过周期性自动存档。默认不开启该开关，不改变正式玩法。

## 当前存档位规则

- 当前固定 `3` 个故事存档位
- 每个存档位相互独立
- 继续游戏默认基于最近使用的存档位

## 继续游戏规则

- 强退或中途返回主菜单后，允许继续当前战斗
- 继续目标通常回到 `main.tscn`
- 如果没有战斗存档但有主线存档，则回到主线准备流程

## 通用 UI 组件

本轮 UI 重构抽出了这些通用组件：

- `scripts/ui/theme/survivors_ui_theme.gd`：幸存者风格颜色、按钮、卡片、字体缩放、弹窗尺寸常量。
- `scripts/ui/core/survivors_modal.gd`：全屏遮罩 + 居中弹窗，支持按窗口比例和最大/最小尺寸自适应。
- `scripts/ui/components/survivors_card_list.gd`：带常显滚动条的卡牌列表 / 网格。
- `scripts/ui/components/survivors_hover_detail.gd`：悬停详情浮层，按内容自动估算尺寸，避免重复追加三英雄数值段落。
- `scripts/ui/components/survivors_slot_card_factory.gd`：存档位、难度等奖励卡片样式工厂。
- `scripts/game/blessing_unlock_notice_flow.gd`：祝福材料被技能解锁/进化配方锁定时的战斗内短暂提示弹窗。

交接原则：阻塞式选择弹窗优先复用 `SurvivorsModal`；战斗内短提示可复用 `SurvivorsTheme` 生成轻量弹窗；新增卡牌列表优先复用 `SurvivorsCardList`，不要重新写一套固定像素弹窗。
