# 战斗系统与角色机制

## 基础操作

- `WASD`：移动。
- 鼠标：鼠标跟随模式下控制攻击方向。
- `TAB`：切换自动攻击 / 鼠标跟随。
- `Q` / `E`：切换上一名 / 下一名角色。
- `R`：释放当前角色大招。
- `C`：打开角色面板。
- `ESC`：暂停。

## 队伍结构

- 当前战斗固定三名角色：剑士、枪手、术师。
- 同一时间只有一名角色站场。
- 未站场角色通过后台攻击和切人收益参与战斗。
- 角色面板用于查看不同角色的数值、祝福、技能和道具。

## 三角色定位

- 剑士：近战压制、生存、范围斩击。
- 枪手：远程弹幕、持续输出、射程覆盖。
- 术师：范围轰炸、区域控制、AOE 压力。

## 战斗循环原则

- 普攻是持续输出基础。
- 切人是节奏转换和站场角色切换。
- 大招是高价值资源，不应该廉价连发。
- 祝福改变角色数值、技能持续时间、技能数量、连段次数和技能解锁。

## 祝福与技能

- 普通升级提供祝福三选一。
- 角色绑定祝福三人共用。
- 技能绑定祝福用于技能解锁和技能加成。
- 当前祝福技能会显示在底部技能栏，并有独立冷却。
- 剑刃风暴、无限装填、波涛汹涌等技能由祝福需求解锁，不再由旧卡牌或图谱解锁。

## 英雄特性训练

英雄特性训练仍可作为升级菜单的一部分存在，用于强化当前队伍角色的基础倾向：

- 剑士特性：最大生命、生命恢复、剑士普攻伤害和相关入场收益。
- 枪手特性：移动速度、闪避、枪手普攻伤害和相关入场收益。
- 术师特性：大招能量恢复、拾取范围、术师普攻伤害和相关入场收益。
- 共同致富：提高队伍特性并缩短切人冷却。

## 开发者模式

开发者模式用于快速验证：

- 角色升级。
- Boss / 小 Boss 生成。
- 忽略伤害。
- 无冷却。
- 性能指标。

开发者模式应继续使用祝福升级入口。
