# 战斗系统与角色机制

## 基础操作

- 移动：`WASD`
- 攻击方向：鼠标
- 切换角色：`Q` 上一位，`E` 下一位
- 释放大招：`R`
- 暂停：`ESC`
- 高级信息切换：``~`` 键

## 队伍结构

- 每次战斗为 3 人小队
- 始终只有 1 名角色站场
- 其他角色通过后台攻击、协同效果、切人收益参与战斗
- 队伍顺序具有真实意义：
  1 号位决定开场站场角色
  轮转顺序与 `Q/E` 对应

## 三角色定位

### 剑士

- 定位：近战压制、生存、近身收益
- 演出关键词：月牙斩、重斩、交叉斩、近战范围压迫
- 当前特征方向：
  不鼓励远程化
  更适合吸血、减伤、正面压制、近战收益

### 枪手

- 定位：远程 poke、持续点杀、拉扯
- 演出关键词：多发子弹、锁定、弹幕、轨迹覆盖
- 当前特征方向：
  强调射程、命中效率、远距离输出和满屏子弹感

### 术师

- 定位：AOE、区域覆盖、控制
- 演出关键词：轰炸、减速、爆炸、区域牵引
- 当前特征方向：
  攻速较慢，但范围和控制力强

## 战斗循环原则

- 普攻是持续输出基础
- 切人是节奏转换和收益放大器
- 大招是最珍贵的高强度资源
- 理想体验：
  普攻维持场面
  切人制造收益与高光
  大招作为关键时机的终极压制

## 切人设计目标

- 切人必须有明显收益
- 玩家应当明显感受到：
  切人不是“换个角色继续打”
  而是“触发一个新的战术节奏点”
- 进场技、退场技、后台输出、本体 Build 共同为切人提供价值

## 大招设计原则

- 不应廉价连发
- 伤害、画面、释放门槛都要明显高于普通攻击
- 大招释放后必须有明确演出，不允许“看起来像打到但没感觉”

## 英雄特性训练

当前升级训练选项使用英雄特性而不是三维属性：

- 剑士特性：最大生命、生命自动恢复、剑士普攻伤害。
- 枪手特性：移动速度、闪避率、枪手普攻伤害。
- 术师特性：大招能量自动恢复、吸取范围、术师普攻伤害。
- 共同致富：三名英雄特性各 +0.35，并让切换英雄冷却时间乘算 ×90%。

## 当前战斗内主要要素

- 角色普攻
- 角色进场技
- 角色退场技
- 角色大招
- 后台协同攻击
- 经验晶体
- 回复爱心
- 升级 Build
- 精英 / Boss 压力

## 开发者模式

当前 `developer_mode.gd` 提供的核心能力：

- 视为开发者模式开启
- 只刷 Boss
- 忽略伤害
- 升级时提供全部 Build 卡
- 禁用正式存档影响
- 大招可自由释放
- 可快速补满当前升级经验

用途：

- 测试角色特效
- 测试构筑完整形态
- 测试 Boss 演出
- 快速验证 UI 与技能问题
