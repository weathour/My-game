# 敌人、Boss、关卡与难度

## 主关卡时间轴目标

- 设计目标：主关卡 8 分钟进入 Boss
- 当前工程实现：总时长 `600s`，Boss 在 `480s` 出场
- 可以理解为：
  前 8 分钟为成长期
  后 2 分钟为 Boss 战阶段

## 当前普通怪设计方向

1. 基础追击怪
   最早的蓝色怪，负责基础压迫

2. 远程发弹怪
   增加弹幕压力

3. 肉且定时加速怪
   负责逼迫走位与拆火

4. 脆皮冲锋海怪
   一次大量刷新，制造短时压迫

5. 肉且定时冲刺怪
   负责切入和突然贴脸

## 精英怪方向

- 精英怪不单独重新发明体系
- 以普通怪的双特性组合为主
- 例如：
  远程 + 加速
  肉 + 冲刺
  追击 + 发弹

## 当前精英奖励状态

- 旧版精英奖励已临时移除
- 当前版本不再掉落旧的精英奖励内容
- 后续需要重做一套新的精英奖励体系

## Boss 方向

Boss 设计核心：

- 不是纯血厚
- 而是通过分阶段弹幕与技能组合制造演出和压力

### Boss UI 目标

- Boss 出现后，顶部倒计时隐藏
- 改为显示 Boss 名字和超长血条
- 血条跨度应覆盖左侧信息区到右侧队伍区

### Boss 战目标时长

- 在 8 分钟左右已成型的角色下
- Boss 理想存活时间约 2 分钟

### Boss 三阶段方向

#### P1

- 高频率全角度发射常规弹幕
- 高频率发射正弦轨迹弹幕
- 正弦弹视觉只表现出 1/4 周期感

#### P2

- 保留 P1 手段
- 正弦弹飞行到一定阶段后被吸收并分裂
- 追加多方向激光攻击

#### P3

- 继承 P2 并提高频率
- 增加巨大能量球与分裂弹
- 增加孔雀开屏式扇形填充弹幕

## Boss 视觉要求

- Boss 子弹必须和：
  角色攻击
  经验晶体
  地面掉落物
  明显区分

## 难度模式共识

### 不变项

- 精英出场时间固定
- Boss 出场时间固定
- 关卡主流程固定

### 变化项

- 小怪组合权重
- 生存容错
- 资源供给
- Boss 技能层数与联动复杂度

## 难度模式方向

### 故事模式

- 目标：稳定体验角色和构筑
- 容错高
- 小怪生态简单
- Boss 保留核心机制但压力较低

### 普通模式

- 当前默认基准盘
- 用于平衡和调参

### 困难模式

- 更强调切人、走位、执行
- 提高危险怪组合频率
- Boss 在普通基础上增加机制联动层

### 地狱模式

- 面向硬核挑战
- 小怪组合更高压
- Boss 机制最完整
- 惩罚只作用于当前地狱存档，不污染其他模式

## 当前设计结论

- 模式区分不能只靠倍率
- 小怪生态差异 + Boss 机制差异，才是主要难度来源

## 基于 Build/敌人压力模型的四档难度方案

当前无尽难度入口已经有四个难度 id：

```text
easy / normal / hard / hell
```

新的难度设计不应只是“敌人血量乘以一个数字”。难度应该是敌人压力时间线与 Build 覆盖能力之间的关系：

```text
DifficultyRisk(route, t, difficulty)
  = UncoveredPressure(EnemyPressure(t, difficulty), BuildCoverage(route, t))
```

其中 `EnemyPressure` 拆为：

```text
density / durability / contact / ranged / burst / mobility / control / boss
```

四档难度的设计目标如下：

| 难度 | 目标体验 | 目标风险带 | 对 Build 的要求 |
| --- | --- | --- | --- |
| 简单 easy | 学习、试卡、允许乱选 | 合理路线 risk ≈ 0.35-0.50 | 单主轴或散选也能体验到成型感 |
| 普通 normal | 基准平衡盘 | 合理路线 risk ≈ 0.50-0.63 | 主轴清晰即可，副轴不强制 |
| 困难 hard | 检验路线连贯性 | 合理路线 risk ≈ 0.63-0.76 | 12 级前应形成主轴，18 级前要补一个压力答案 |
| 地狱 hell | 高压挑战 | 强路线 risk ≈ 0.76-0.90 | 需要主轴、副轴和敌人压力答案，但不能锁死唯一最优解 |

初始参数建议只作为模型校准起点，不是最终数值：

| 参数 | 简单 | 普通 | 困难 | 地狱 | 说明 |
| --- | ---: | ---: | ---: | ---: | --- |
| spawn_interval_scale | 1.12 | 1.00 | 0.90 | 0.78 | 越小刷怪越密 |
| enemy_health_scale | 0.85 | 1.00 | 1.15 | 1.35 | 提高耐久压力 |
| enemy_damage_scale | 0.80 | 1.00 | 1.10 | 1.22 | 提高生存压力 |
| enemy_speed_scale | 0.95 | 1.00 | 1.04 | 1.08 | 小幅提高机动压力，避免手感失真 |
| special_pressure_scale | 0.85 | 1.00 | 1.15 | 1.35 | 远程、冲刺、炮台、特殊组合权重 |
| boss_pressure_scale | 0.90 | 1.00 | 1.10 | 1.25 | Boss 血量、技能层数、技能频率的综合压力 |
| recovery_margin_scale | 1.15 | 1.00 | 0.95 | 0.90 | 掉心、容错或恢复窗口，不直接削经验 |

调参原则：

```text
1. 简单难度降低 burst/contact/damage 多一些，保留升级和成型乐趣。
2. 普通难度作为所有 Build 首先校准的基准盘。
3. 困难难度不只加血，应提高危险组合出现率，让偏科 Build 暴露短板。
4. 地狱难度重点增加多轴压力：密度 + 远程 + 爆发 + Boss，而不是单纯血牛化。
5. 不建议用大幅降低经验作为难度手段，因为这会破坏 Build 系统的核心乐趣。
```

四档难度的验收方式：

```text
对每个难度运行多条路线：
- 三英雄均衡
- 剑士主轴
- 枪手主轴
- 术师主轴
- 召唤支援
- 奶推/生存支援
- 中期转向
- 低协同散选

检查：
- 简单：低协同散选不能早早崩盘。
- 普通：合理路线都在接近风险带内。
- 困难：低协同散选明显弱，合理路线仍有多种答案。
- 地狱：强路线也有高压窗口，但不能出现唯一最优解。
```

当前首批落地：

```text
- scripts/game/difficulty_profile.gd 定义 easy / normal / hard / hell 四档 profile。
- UI 四档全部可选，存档 difficulty 会规范化保存。
- 无尽模式会把 difficulty_profile 接入刷怪间隔、敌人血量、伤害、速度、危险怪权重、特殊怪攻击频率和 Boss 攻击压力。
- 敌人压力模型也能按 difficulty_id 输出不同压力，用于后续完整时间线评估。
```

性能保护原则：

```text
- difficulty_profile 同时携带 active_enemy_limit / enemy_projectile_limit / temporary_effect_limit。
- 普通刷怪会被敌人上限裁剪，避免高难度密度直接造成节点爆炸。
- 敌方弹幕生成前检查 enemy_projectile_limit，分裂弹也会按剩余容量裁剪。
- 高频扫描型敌人行为应按固定间隔采样，不每帧遍历大组。
```

### 地狱难度二次调优

首批实装后地狱难度偏低，因此本轮把地狱从“略高压”改为真正的高压挑战：

- 刷怪间隔系数从 `0.78` 下调到 `0.62`。
- 最低刷怪间隔系数也下调到 `0.62`，避免后期被普通难度同一个最低间隔上限卡住。
- 普通敌人生命从 `1.35` 提高到 `1.62`。
- 精英/小 Boss 生命从 `1.42` 提高到 `1.82`。
- Boss 生命从 `1.25` 提高到 `1.55`。
- 敌人伤害从 `1.22` 提高到 `1.48`。
- 危险怪、远程怪、爆发怪权重显著提高。
- 特殊怪攻击间隔从 `0.84` 下调到 `0.68`，弹幕数量系数从 `1.15` 提高到 `1.42`。
- Boss 攻击压力从 `1.25` 提高到 `1.65`。

验收标准也同步提高：18 级敌人压力模型中，地狱总压力至少应比普通高 55%。
