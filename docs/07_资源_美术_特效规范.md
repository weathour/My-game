# 资源、美术与特效规范

## 建议规格

- 游戏基准分辨率：`1920x1080`
- 角色：`256x256`
- 小怪：`128x128` 或 `256x256`
- Boss：`512x512`
- 子弹 / 小图标：`64x64` 或 `128x128`
- 普通特效：`256x256`
- 大招特效：`512x512` 到 `1024x1024`
- UI 图标：`64 / 128 / 256`
- 按钮：`320x120` 或 `420x140`
- 面板：`512x256` 或 `1024x512`
- 祝福 / 奖励卡片：`420x220`

## 资源格式

- 角色、特效、UI 优先使用 `PNG`。
- 背景大图可使用 `JPG` 或 `PNG`。
- 文字尽量和底图分离，便于后续修改和本地化。
- 不提交 `.DS_Store`、`__MACOSX/`、`._*` 等操作系统或压缩包元数据；这些文件不参与运行且会污染素材目录。

## 特效原则

- 特效必须服务玩法辨识，不只是好看。
- 玩家应能一眼看出是谁的技能、打哪里、和其他技能有什么不同。
- 特效范围尽量贴近真实伤害范围。
- 不允许“看起来打到了但没伤害”。
- 不允许“没碰到却判定中了”。
- 如果特效和判定冲突，优先调整判定贴合特效。

## 替换原则

- 替换资源时优先保持逻辑接口不变。
- 不因为换图就重写整套判定。
- 只有新特效表达方式完全变化时，才调整对应判定逻辑。

## 当前阶段

- 当前大量素材仍是测试贴图或测试特效。
- 现阶段目标是可替换、可运行、可验证观感。
- BGM 是非商用占位素材，发布前需要替换或完成授权确认。
