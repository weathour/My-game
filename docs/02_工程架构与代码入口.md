# 工程架构与代码入口

## 基本信息

- 引擎：Godot 4.6
- 主运行场景：`res://scenes/main_menu.tscn`
- 配置文件：`project.godot`

## 目录结构

- `assets/`：测试图片、音频和临时素材。
- `effects/`：剑士、枪手、术师技能特效场景。
- `scenes/`：主菜单、战斗场景、HUD、玩家、敌人、拾取物等场景。
- `scripts/`：运行逻辑。
- `scripts/game/`：战斗场景 flow，负责刷怪、奖励、HUD、地图、存档和 session。
- `scripts/player/`：玩家模块，负责角色、祝福、技能、普攻、大招、数值、存档和奖励应用。
- `scripts/enemies/` / `scripts/enemy/`：敌人行为、属性、状态、掉落、Boss 行为。
- `scripts/ui/`：通用 UI、HUD 面板、升级详情、存档界面。
- `docs/`：当前项目文档。

## 主要场景

- `scenes/main_menu.tscn`：主菜单。
- `scenes/main.tscn`：战斗主场景。
- `scenes/player.tscn`：玩家节点。
- `scenes/enemy.tscn`：敌人节点。
- `scenes/hud.tscn`：战斗 HUD。
- `scenes/level_up_ui.tscn`：升级和奖励菜单。
- `scenes/pause_menu.tscn`：暂停菜单。

## 主要脚本职责

- `scripts/main.gd`：战斗主控和 flow 装配，不应继续堆具体玩法细节。
- `scripts/player.gd`：玩家节点协调层，保留必要状态和桥接函数；新增玩法优先放入 `scripts/player/*` 模块。
- `scripts/player/player_blessing_system.gd`：祝福定义、发牌、应用、合成规则。
- `scripts/player/player_blessing_skill_state.gd`：祝福驱动的技能解锁、技能数量、持续时间和连段倍率。
- `scripts/player/player_level_flow.gd`：升级选项、英雄特性训练、小 Boss 奖励入口。
- `scripts/player/player_upgrade_applier.gd`：当前奖励分发器，只处理祝福、装备、小 Boss 训练、最终核心和空选项。
- `scripts/player/player_equipment_flow.gd`：角色道具持有、转赠和装备数值。
- `scripts/player/roles/*_role.gd`：三名角色的普攻和角色差异。
- `scripts/abilities/*_ability.gd`：祝福解锁后的独立技能。
- `scripts/game/reward_flow.gd`：升级 UI、小 Boss 奖励、最终奖励的流程桥接。
- `scripts/game/enemy_spawn_flow.gd`：地图边界内波次刷怪和警告。
- `scripts/game/game_hud_flow.gd`：HUD 数据投影。
- `scripts/ui/hud/character_panel.gd`：角色面板、祝福显示、祝福合成、道具显示和转赠入口。

## 架构护栏

- 不把新玩法直接堆进 `player.gd` 或 `main.gd`。
- 新系统先判断归属：`scripts/player/*`、`scripts/game/*`、`scripts/ui/*`、Autoload，或纯数据 helper。
- 祝福、技能、装备、刷怪、HUD、角色面板必须保持模块边界。
- 升级和技能解锁以祝福体系为准，不要新增废弃进度系统入口。
- `scripts/tests/check_architecture_contract.py` 用于检查核心边界。
