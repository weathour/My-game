# 当前实现状态与后续重点

## 当前阶段判断

当前项目处在 **新 Build 大改的 Alpha 可玩实现阶段**：核心框架、首批卡池、三英雄专属、Boss 三选一奖励、技能图谱、难度模型、升级曲线、发牌刷新、开发者模式同步和自动回归检查都已落地。下一阶段重点不是继续堆大框架，而是做实机路线验证、数值调参、内容补量和 UI/演出 polish。

## 当前已经基本具备的内容

- 本地成就系统、成就提示弹窗，以及后续 Steam 成就适配边界
- 窗口/全屏显示设置，窗口模式下固定 16:9 比例
- 地图边界显示、玩家边界限制、右下角小地图
- 统一幸存者风格 UI 组件：弹窗、卡牌列表、悬停详情、卡片工厂
- 主菜单设置面板居中/自适应修复
- 升级界面卡牌短摘要 + 悬停详情，大招/普攻/独立被动详情说明
- 普通升级 Build 已接入首批三槽位发牌：主轴延续 / 联动共鸣 / 转向补强，并提供每次升级一次发牌刷新；升级 UI 统一显示三张 Build 卡，不再按槽位分区。
- 首批 35 张候选卡与 3 个成型节点已有纯模型、运行时格式化、基础战斗效果承载和 smoke 测试；6 级队伍里程碑已接入实战反馈、底部冷却槽和三英雄差异触发；旧主题式 Build 池仅保留为兼容/历史链路。
- 三张 12 级英雄专属已接入独立 ability：`剑影留形`、`侦察无人机`、`守护傀儡` 分别拥有独立冷却、技能栏显示、存档冷却和专属战斗效果。
- Boss Build 奖励已改为三选一英雄定向增强：剑士/枪手/术师各一张，优先提升对应专属卡，专属满级后转为同英雄溢出补强。
- 开发者模式已同步首批 Build：升级弹窗、开发者卡牌列表和一键发卡都走新卡池门槛。
- 战斗 HUD 已新增可打开的 Build 技能图谱，用轻量自绘的单字小方块展示所有首批 Build 的演进可能；同等级技能固定在同一横排，悬停显示完整技能说明，并支持默认主干视图、路线切换、关系筛选、搜索定位和点击锁定局部前置/后续。
- 英雄特性训练与共同致富：按当前队伍生成特性成长，强化对应英雄入场/离场效果，并提供切人冷却乘算缩减。
- 四档无尽难度 easy / normal / hard / hell 已接入敌人血量、伤害、刷怪密度、危险怪权重、特殊怪弹幕、Boss 压力和性能上限裁剪；地狱难度已做二次上调。

## 当前已验证的主爽点

- 切人节奏成立
- 大招演出能形成明显高潮
- 三个角色已有基本差异
- 6 级首个 Build 质变已有可见反馈
- Boss 掉落能让玩家在三个英雄方向中明确选择一次中后期强化
- 项目已经脱离“只有概念没有手感”的阶段

## 当前主要短板

### 构筑深度仍然需要继续扩展

- 首批 Build 已经进入升级局内循环，6 级有实战里程碑，12 级三张专属也已拆成独立 ability；18/25 的路线完全成型感仍需要补更强的演出、机制链和敌人互动。
- `剑影留形`、`侦察无人机`、`守护傀儡` 已有专属实现，但仍应继续提升视觉辨识度、召唤物行为丰富度和与敌人压力的对应关系。
- 新增卡必须同步补齐卡牌向量、等级上限、发牌门槛、UI 详情、战斗效果和 smoke 验证。

### 构筑与角色特征仍需继续绑定

- 三个角色必须继续强化各自的独特演出语言
- 剑士应该更像夸张斩击体系
- 枪手应该更像夸张弹幕体系
- 术师应该更像轰炸和区域控制体系

### 工程复杂度已经明显上升

- `player.gd` 和 `enemy.gd` 仍然是高风险枢纽
- 本轮已进一步抽出 `scripts/player/player_theme_skill_flow.gd` 和 UI 通用组件，但 Build 触发、普攻、大招、存档仍需要谨慎验证
- 继续堆功能会降低后续维护效率

## 当前适合优先推进的方向

1. 用 5 条标准路线做实机验证：剑士主轴、枪手主轴、术师主轴、剑士转枪手、三英雄均衡协同。
2. 继续加强 6/12/18/25 的体感，特别是入场/离场、共鸣、大招质变和 Boss 三选一奖励的演出辨识度。
3. 继续打磨三张专属 ability 的视觉和敌人交互，让剑影、无人机、傀儡不只是数值补强。
4. 完善主线准备界面和故事模式骨架。
5. 继续丰富怪物生态和 Boss 剧本，让不同 Build 路线面对不同敌人压力时有不同答案。
6. 设计第 4、第 5 个角色前，沿用当前队伍角色表，并补齐新英雄的 trait 声明、卡牌文案和战斗效果。

## 当前不建议立刻做的事情

- 大规模重构全部代码
- 一口气做完整剧情演出系统
- 一开始就做最终美术成品规格
- 把所有未验证系统同时并行推进

## 推荐的下一阶段开发顺序

1. 稳住当前战斗可玩版本，并记录上述 5 条标准路线的实机体验。
2. 优先调强质变体感：6 级要明显，12 级专属要明显，Boss 三选一要明显。
3. 补路线识别 UI：让玩家一眼知道当前偏主轴、副轴、转向还是补短板。
4. 继续打磨 Build 深度与角色差异。
5. 把主线准备和关卡推进流程跑通。
6. 继续扩展关卡内容和难度模式。
7. 再进入更系统的剧情、营地互动和长期养成。

## 结论

- 项目已经不是“做不出来”的阶段
- 当前核心任务不是从零开始想，而是把现有系统持续整理、扩展、去同质化
- 这套文档的目的，就是让后续继续开发时不需要反复从聊天记录里捞结论

## 本轮交接重点（2026-05-02）

- `scripts/build/build_first_batch_database.gd` 是首批新 Build 卡牌、路线标签、等级门槛、能力包关系和卡面字段的权威数据源。
- `scripts/build/build_first_batch_model.gd` / `scripts/build/build_first_batch_runtime.gd` 负责从当前队伍状态生成三张升级候选、一次刷新和转向/联动推荐。
- `scripts/build/build_database.gd` 与 `scripts/build/build_system.gd` 主要保留旧主题、Boss Build 奖励、小 Boss 奖励和兼容链路；普通升级不再以旧主题树作为主流程。
- `scripts/player/player_attribute_flow.gd` 与 `scripts/player/roles/role_attribute_rules.gd` 负责英雄特性训练、旧存档迁移、进出场特性加成、共同致富计数和切人冷却倍率。
- `scripts/player/player_first_batch_card_applier.gd` 负责把首批 Build 卡转换成角色属性、special key、冷却槽和战斗收益。
- `scripts/player/player_first_batch_milestone_flow.gd` 负责队伍 6/12/18/25 级里程碑。
- `scripts/abilities/swordsman_blade_shadow_ability.gd`、`scripts/abilities/gunner_spotter_drone_ability.gd`、`scripts/abilities/mage_guardian_puppet_ability.gd` 是三张 12 级专属独立冷却 ability。
- `scripts/player/player_theme_skill_flow.gd` 负责旧主题卡触发型被动和独立冷却被动。
- `scripts/player/player_skill_cooldown_flow.gd` 与 `scripts/player/player_ultimate_flow.gd` 负责把普攻/独立被动/大招强化投影到 HUD 悬停说明。
- `scripts/ui/build_graph/*` 与 `scripts/build/build_skill_graph_model.gd` 负责 Build 技能图谱；更新首批卡关系时要同步检查图谱节点和边。
- `scripts/game/difficulty_profile.gd` 与 `scripts/build/build_enemy_pressure_model.gd` 负责四档难度和敌人压力评估。
- `scripts/ui/*` 下的通用组件已经成为升级界面、暂停/结算、存档/难度、主菜单设置和 Build 图谱的共享 UI 基础。
- 已新增/扩展 smoke：`ui_build_handoff_smoke.gd`、`player_level_curve_smoke.gd`、`build_first_batch_model_smoke.gd`、`build_first_batch_balance_smoke.gd`、`build_first_batch_runtime_smoke.gd`、`build_enemy_pressure_model_smoke.gd`、`difficulty_profile_smoke.gd`。
